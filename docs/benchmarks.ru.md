# Бенчмарки

!!! warning "Читайте методологию"
    Бенчмарки легко подделать. Здесь — **in-process ASGI-диспетчеризация**:
    изолируем *накладные расходы фреймворка* (роутинг + разбор запроса +
    построение ответа), вызывая ASGI `__call__` напрямую, **без сети**. В
    продакшене доминируют HTTP-транспорт (`uvicorn`) и работа ваших
    хендлеров/БД; оверхед фреймворка — малая доля. Числа воспроизводимы скриптом
    ниже.

## Методология

- Каждое приложение отдаёт статический маршрут (`GET /v1/ping` → `{"pong": true}`)
  и динамический (`GET /v1/item/{id}`).
- Вызываем ASGI-callable `N` раз после прогрева на 1 000 итераций, меряем
  запросы/сек и мкс/запрос.
- Сравниваем с **FastAPI/Starlette** с эквивалентными endpoint'ами.

Запуск:

```bash
pip install endocore fastapi
py -3 benchmarks/bench.py 50000
```

## Результаты

Машина: **Python 3.13.2 · Windows · AMD64** (ваши числа будут другими; важнее
соотношения, чем абсолюты).

| Маршрут | EndoCore | FastAPI/Starlette | Соотношение |
|---------|---------:|------------------:|:-----------:|
| Статика  `GET /v1/ping`      | **≈ 23 700 req/s** | ≈ 29 700 req/s | 0.80× |
| Динамика `GET /v1/item/{id}` | **≈ 24 100 req/s** | ≈ 22 500 req/s | **1.07×** |

**Выводы (честно):**

- На тривиальном статическом маршруте гипероптимизированный путь ответа FastAPI
  немного быстрее (~0.8×).
- На **динамических маршрутах** trie-резолвер EndoCore чуть **быстрее** (~1.07×).
- Оба фреймворка одного порядка. На таких скоростях (десятки тысяч req/s
  *чистой диспетчеризации*) фреймворк **никогда** не бутылочное горлышко — им
  становятся БД, сериализация и сеть.

## Почему EndoCore конкурентоспособен

- Дерево маршрутов — **trie, построенный один раз при старте** и кэшированный;
  резолв — один проход, без regex на каждый запрос.
- Частый случай `handler(request)` использует **быстрый путь** без разрешения
  зависимостей.
- `Request` — тонкий объект над сырым ASGI scope; заголовки/query парсятся один
  раз, тело — лениво.

## Чего это **не** меряет

- Реальную сетевую пропускную способность (это `uvicorn`, общий для обоих).
- Сериализацию больших JSON.
- Латентность БД (реальное горлышко в большинстве приложений — используйте
  [async ORM](orm/async.md), чтобы не блокировать event loop).
