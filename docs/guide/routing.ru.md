# Роутинг

Роутер — сердце EndoCore. Здесь нет таблицы маршрутов, которую нужно вести —
**таблица маршрутов и есть дерево папок**.

## Правила

| Файловая система | URL |
|-----------|-----|
| `Api/v1/User/Role/Get.py` | `GET /v1/user/role` |
| `Api/v1/User/Role/Post.py` | `POST /v1/user/role` |
| `Api/v1/User/[id]/Get.py` | `GET /v1/user/42` → `id = "42"` |
| `Api/v2/User/Role/Post.py` | `POST /v2/user/role` |

- **Папка = сегмент URL.** Имена папок при сопоставлении приводятся к нижнему
  регистру (`User` → `user`), так что на диске можно держать PascalCase.
- **Имя файла = HTTP-метод.** `Get.py`, `Post.py`, `Put.py`, `Patch.py`,
  `Delete.py`, `Head.py`, `Options.py`. Нормализуется через `.upper()`.
- **Папка `[name]` = динамический сегмент**, попадает в `request.path_params`.
- **Первый сегмент `vN` = версия** (см. [Версионирование](versioning.md)).

## Контракт обработчика

Каждый файл endpoint'а определяет `handler`:

```python
from endocore import Request, Response

async def handler(request: Request) -> Response:
    return Response.json({"ok": True})
```

- `handler` может быть `async def` или обычным `def`.
- Может принимать только `request`, а может использовать
  [внедрение зависимостей](dependency-injection.md) для дополнительных параметров.
- Возвращает `Response`, либо `dict`/`list` (→ JSON 200), `str` (→ текст),
  `None` (→ 204) или `(content, status[, headers])`.

Опциональная `def init(): ...` выполняется один раз при старте — для настройки.

## Статика важнее динамики

Если запросу подходят и статическая папка, и папка `[id]`, побеждает
**статическая**:

```text
Api/v1/User/Me/Get.py       # GET /v1/user/me   -> статический обработчик
Api/v1/User/[id]/Get.py     # GET /v1/user/42   -> динамический обработчик
```

## Результаты резолва

- **200** — маршрут найден.
- **404** — нет такого пути или версии (запрос без версии по умолчанию — 404).
- **405** — путь существует, но не для этого метода (с заголовком `Allow`).

## Файлы, не являющиеся маршрутами

Файлы, чьё имя не является HTTP-методом, — **не** endpoint'ы, а обычные модули
(сервисы, хелперы). Файлы внутри папки `Services/` никогда не считаются
маршрутами.

## Просмотр маршрутов

```bash
end routes      # каждый метод + URL + файл, на который он указывает
end check       # дубли маршрутов, битые обработчики, пропущенные файлы
```

## Default-to-latest (по желанию)

По умолчанию запрос без префикса `vN` — это 404 («явное лучше неявного»).
Можно включить резолв путей без версии в самую свежую версию:

```bash
end dev --default-version latest
```

Каждый такой запрос **логируется** с версией, которой он был обслужен.
